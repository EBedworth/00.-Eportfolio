Exploring Linters

In Unit 6, we were asked to run some Python tools which check our code for linters (stylistic errors) and comment on the output of these tools, focusing on how useful this output was to improving our code.

Testing Tools

Two tools were to be used for testing our example code during this exercise. The tools were:

  • PyLint
  • Flake8

    • Were provided with some sample Python files to test our code, and asked to comment on the output from our testing tools and then to fix these files.

    styleLint.py

    Our first file was styleLint.py, formatted as below: 

    
def factorial(n):
""" Return factorial of n """
if n == 0:
return 1
else:
return n*factorial(n-1)
    When this code was run with Python, an indentation error was returned. In order to fix this, the indentation was corrected as follows: 

    
def factorial(n):
""" Return factorial of n """
if n == 0:
	return 1
else:
	return n*factorial(n-1)

    Pylint and Flake8 Comparison

    The output from PyLint was more detailed than the output from Flake8, which would allow us to improve our code to a greater degree. Some errors were omitted from Flake8, such as the 'invalid-name' errors which are shown for the global variables in the file. However, the PyLint error for these global variables is not necessarily correct (and is not correct in this case) as PyLint is considering all of the variables outside of the main body to be constants. Some the variables in this file are not constants, but are still globals.

    In addition, Flake8 did not detect the style of the filename, the missing docstring, or the redefined outer name.

    It appears to me that PyLint is perhaps more detailed in the checks it performs, however, I would expect that each error received from PyLint should be scrutinised by the programmer, rather than immediately changed. There are some examples of the output above which should not be changed, based on the output of PyLint. While Flake8 is less detailed, this perhaps means that the changes suggested are obvious and should require less scrutiny.

    Corrected Files

    Following the output from both testing tools, I proceeded to correct the file as required. 

    
""" # SOURCE OF CODE: https://docs.pylint.org/en/1.6.0/tutorial.html
"""
import string

SHIFT = 3
CHOICE = raw_input("would you like to encode or decode?")
WORD = (raw_input("Please enter text"))
LETTERS = string.ascii_letters + string.punctuation + string.digits
ENCODED = ''
if CHOICE == "encode":
    for letter in WORD:
        if letter == ' ':
            ENCODED = ENCODED + ' '
        else:
            x = LETTERS.index(letter) + SHIFT
            ENCODED = ENCODED + LETTERS[x]
if CHOICE == "decode":
    for letter in WORD:
        if letter == ' ':
            ENCODED = ENCODED + ' '
        else:
            x = LETTERS.index(letter) - SHIFT
            ENCODED = ENCODED + LETTERS[x]

print(ENCODED)
    Note that for the above, I changed the file to satisfy the requirements of PyLint, however, there are now some capitalised non-constant variables. This is not a good choice stylistically, and so should not have been implemented.

Codio Activity - Producer-Consumer Mechanism

In Unit 3, we were asked to answer some questions around the producer-consumer mechanism, specifically focused on a block of code we were provided.

Originating Code


# code source: https://techmonger.github.io/55/producer-consumer-python/
 
from threading import Thread
from queue import Queue
 
q = Queue()
final_results = []
 
def producer():
    for i in range(100):
        q.put(i)
        
 
def consumer():
    while True:
        number = q.get()
        result = (number, number**2)
        final_results.append(result)
        q.task_done()
   
   
for i in range(5):
    t = Thread(target=consumer)
    t.daemon = True
    t.start()
    
producer()
 
q.join()
 
print (final_results)

Questions

How is the queue data structure used to achieve the purpose of the code?

The queue is used in the code to provide an iterable object with which to eventually build a list containing numbers and their corresponding squared value. The code populates the queue with numbers in the producer function, then creates a tuple of the number and its square, which is finally appended to final_results.


What is the purpose of q.put()?

q.put(i) is used to add an item to the queue and also increment the count of unfinished tasks which the queue has to process (used later with q.join()).


What is achieved by q.get()?

q.get() returns and removes an item from the queue, which is then added as the first element of the tuple result, and squared to become the second element of result. q.get() does not decrease the count of unfinished tasks for the queue, but this is achieved through the use of q.task_done() in the same loop.


What functionality is provided by q.join()?

q.join() is used to prevent the program proceeding until all queue tasks have been completed. In this instance, this is until the loop in the consumer function has completed.


Extend this producer-consumer code to make the producer-consumer scenario available in a secure way. What technique(s) would be appropriate to apply?

Making use of the parameters of Queue, get and put would increase the security of the code. q could be implemented with a maximum size, to prevent unnecessarily large amounts of memory being consumed by continuously adding items or adding large data sets to the queue. For get and put, making use of the Timeout parameter would also increase security, as this would ensure that code does not hang for an indefinite amount of time should the queue become full.

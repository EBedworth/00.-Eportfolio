Programming Language Concepts - Evil Regex

In Unit 4, we were instructed to read Larson (2018) and Weidman (n.d.) and answer a number of questions on possible threats introduced through the use of Regular Expressions.

Questions

What is ReDOS and what part do ‘Evil Regex’ play?

ReDOS is the term for a denial of service attack carried out by exploiting certain implementations of regular expressions. The exploitation causes the regular expression to use a significant amount of processing resource to evaluate a given input, leaving little or no processing power available to other tasks. ‘Evil Regex’ is the term for a regex pattern which can be exploited in this way, given a suitably structured input.

What are the common problems associated with the use of regex? How can these be mitigated?

Regular expressions can, in the worst case, have exponential time complexity (Roichman & Weidman, 2012). This means that evaluating all possible inputs for a given regular expression is often impractical. This particular issue can be mitigated by using an tool to check regular expression implementation programmatically. One such tool is ACRE (Larson, 2018), which uses a series of checkers to look for certain, known problematic patterns within a given regular expression.

How and why could regex be used as part of a security solution?

Well-designed regular expressions can also be utilised as part of a security solution to prevent certain types of attacks. Regular expressions are particularly useful in filtering out malicious input, where user input is possible to a system. This could even be expanded to filter out strings which might be used to execute a ReDOS attack.

References

Roichman, A. & Weidman, A. (2012) Regular Expression Denial of Service. Available from: https://checkmarx.com/wp-content/uploads/2015/03/ReDoS-Attacks.pdf [Accessed 12 December 2023].

Larson, E. (2018) ‘Automatic checking of regular expressions’, 2018 IEEE 18th International Working Conference on Source Code Analysis and Manipulation (SCAM). Circulo de Bellas Artes, Madrid, 23-24 September. New York City: IEEE. 225-234.

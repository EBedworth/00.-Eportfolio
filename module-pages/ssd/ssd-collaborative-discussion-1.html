Collaborative Discussion 1

Our first collaborative discussion, started in Unit 1, was to select a weakness identified by the Open Worldwide Application Security Project (OWASP) from their Top Ten List and present a flowchart detailing which steps may have led to the weaknesses occurring.

Initial Posting

by Elliot Bedworth - Sunday, 12 November 2023, 1:28 PM

I have selected the number one item from the OWASP Top Ten list to explore: Broken Access Control (OWASP, 2021).

Broken Access Control covers any possible occurrence of access to either data or functionality which should not be possible due to user privilege requirements. The obvious examples of this include a user being able to access the account of another user, or modification of data by a non-authorised user which should be reserved for those with administrator privileges. Beyond this, Broken Access Control also covers users being permitted to access certain data or functionality which is not strictly necessary, known as the principle of least privilege.

I have used an activity diagram to indicate the flow of activity which may occur prior to identifying a failure through Broken Access Control. I believe this is the most suitable way to present this information as it presents a logical sequence of operation, which is easily interpreted, and does not focus on the technical details of how each action might be carried out, which is beyond the scope of this question.

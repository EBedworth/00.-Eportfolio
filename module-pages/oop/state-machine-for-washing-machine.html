State Machine Diagram for Washing Machine

During Unit 3, we were asked to develop a state machine diagram for a washing machine, based on an example state machine diagram from The Unified Modeling Language Reference Manual Second Edition.

The State Machine Diagram

The state machine diagram shows possible states of an object from a given class and shows the possible transitions between these states. This gives an overall picture of way a modelled object should behave during its existence.

The example diagram provided in The Unified Modeling Language Reference Manual Second Edition shows a state machine diagram for a ticket to an event.

Washing Machine

When modelling the washing machine, I identified four states which the system might be in at any one time:

  1. Unlocked - The door is open and the machine is available for use.
  2. Locked - The door is closed and locked and the machine is ready to start.
  3. Running - The machine is performing a wash cycle.
  4. Completed - The machine has finished its last wash cycle.
Whilst there are other states which could be considered, I felt as though these four were enough for mapping the transitions through the user operation of the machine.

For the transitions between each state, I considered the actions which could possibly take place (such as the user stopping the system mid-cycle) and also the actions which must take place (draining the water) before a state could move to another state. For the 'complete' state, the machine must return to 'locked', as the water must still be drained from the system. No input is required for this to occur, only the act of entering into the 'complete' state. Because of this, there is some argument to be made for removing the 'complete' state. I made the decision to leave it in as I believe it will be necessary for implementation to provide indication that a cycle has completed even after the machine has returned to the 'unlocked' state.

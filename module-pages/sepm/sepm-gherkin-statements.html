Gherkin Statements

For Seminar 2, which took place during Unit 2, we were tasked with preparing a sequence of Gherkin statements based on some function, which we could choose from several options.

Gherkin Language

This week, we read the 'Behave - Behaviour Driven Development' documentation, which describes syntax for the Gherkin language. Gherkin is used to capture requirements of different functions in an easily-readable format, which can be shared across both technical and non-technical teams. Gherkin uses very simple keywords to define a pattern of behaviour, given some 'input' actions, explaining what the resulting actions should be.

Gherkin Statements

We were provided with a list of several options, one of which we should develop 3 Gherkin statements to describe interactions with those systems. I chose the option 'Using a new coffee machine', and developed three Gherkin statements to describe the expected behaviour of different functions which might be necessary for a new coffee machine.

Scenario: Running the cleaning cycle for the coffee machine
Given there is no coffee pod inserted into the machine
And there is water in the water tank
And there is a cup under the spout
When the ‘Americano’, ‘Latte’ and ‘Cappuccino’ buttons are held simultaneously for three seconds
Then the cleaning cycle should start
Scenario: Making a cup of coffee
Given there is water in the water tank
And there is a cup under the spout
When the user inserts a coffee pod
And the user selects a drink
Then the machine will make the drink
But if the drink also requires milk
And the milk tank is empty
Then the machine will notify the user to add milk
When the user adds milk
Then the machine will make the drink
Scenario: Refilling the water tank
Given the machine is turned on
And there is no water in the water tank
Then the ‘refill tank’ light will flash
And the machine will not try to make any drink
When the user refills the tank
Then the machine will be able to make drinks again
